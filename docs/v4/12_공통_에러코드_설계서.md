# 12. 공통 에러코드 설계서

> **버전**: v1.0
> **패키지 경로**: `com.example.ticketflow.global.exception`, `com.example.ticketflow.global.common`
>
> **설계 원칙**
>
> | 원칙 | 내용 |
> |------|------|
> | 비즈니스 에러 / 시스템 에러 분리 | `BusinessException` 계층과 미처리 `Exception` 계층을 핸들러에서 구분 |
> | 에러코드 단일 정의 | 모든 에러는 `ErrorCode` enum 한 곳에서 관리. 코드 문자열 하드코딩 금지 |
> | API 응답 코드 일관성 | enum 이름(내부 식별자)과 API 응답 `code` 필드를 분리해 클라이언트 계약 유지 |
> | 401은 Security 필터에서 처리 | JWT 인증 실패는 `@ExceptionHandler` 도달 전에 `JwtAuthEntryPoint`에서 처리 |

## 2. Exception 계층 구조

```java
// global/exception/BusinessException.java  ← 비즈니스 예외 최상위
@Getter
public class BusinessException extends RuntimeException {

    private final ErrorCode errorCode;

    public BusinessException(ErrorCode errorCode) {
        super(errorCode.getMessage());
        this.errorCode = errorCode;
    }

    /** 기본 메시지 대신 동적 메시지를 사용할 때 */
    public BusinessException(ErrorCode errorCode, String detailMessage) {
        super(detailMessage);
        this.errorCode = errorCode;
    }
}
```

```java
// global/exception/NotFoundException.java  → 404
public class NotFoundException extends BusinessException {
    public NotFoundException(ErrorCode errorCode) { super(errorCode); }
}

// global/exception/InvalidRequestException.java  → 400
public class InvalidRequestException extends BusinessException {
    public InvalidRequestException(ErrorCode errorCode) { super(errorCode); }
    public InvalidRequestException(ErrorCode errorCode, String detailMessage) {
        super(errorCode, detailMessage);
    }
}

// global/exception/ForbiddenException.java  → 403
public class ForbiddenException extends BusinessException {
    public ForbiddenException(ErrorCode errorCode) { super(errorCode); }
}

// global/exception/ConflictException.java  → 409
public class ConflictException extends BusinessException {
    public ConflictException(ErrorCode errorCode) { super(errorCode); }
}

// global/exception/HoldExpiredException.java  → 409 (Hold TTL 만료 전용)
public class HoldExpiredException extends ConflictException {
    public HoldExpiredException() { super(ErrorCode.HOLD_EXPIRED); }
}
```

### 계층 구조 요약

```
RuntimeException
└── BusinessException                   ← 모든 비즈니스 예외의 루트 (ErrorCode 보유)
    ├── NotFoundException               → 404
    ├── InvalidRequestException         → 400
    ├── ForbiddenException              → 403
    ├── ConflictException               → 409
    │   └── HoldExpiredException        → 409 (Hold TTL 만료 전용)
    └── (추가 필요 시 확장)
```

> **UnauthorizedException을 별도로 만들지 않는 이유**:
> 401 UNAUTHORIZED는 Spring Security 필터 체인(`JwtAuthEntryPoint`)에서 처리됨.
> 비즈니스 레이어에 도달하기 전에 차단되므로 `@ExceptionHandler` 대상이 아님.

## 3. ErrorResponse DTO

```java
// global/common/ErrorResponse.java
@Getter
public class ErrorResponse {

    private final String code;
    private final String message;
    private final String timestamp;
    private List<FieldErrorDetail> errors;   // @Valid 유효성 실패 시만 포함

    private ErrorResponse(String code, String message) {
        this.code = code;
        this.message = message;
        this.timestamp = LocalDateTime.now().toString();
    }

    /** 단순 비즈니스 에러 */
    public static ErrorResponse of(ErrorCode errorCode) {
        return new ErrorResponse(errorCode.getCode(), errorCode.getMessage());
    }

    /** 동적 메시지 오버라이드 */
    public static ErrorResponse of(ErrorCode errorCode, String detailMessage) {
        return new ErrorResponse(errorCode.getCode(), detailMessage);
    }

    /** @Valid 유효성 실패 — 필드별 에러 목록 포함 */
    public static ErrorResponse ofValidation(List<FieldError> fieldErrors) {
        ErrorResponse response = new ErrorResponse(
            ErrorCode.INVALID_REQUEST.getCode(),
            ErrorCode.INVALID_REQUEST.getMessage()
        );
        response.errors = fieldErrors.stream()
            .map(fe -> new FieldErrorDetail(fe.getField(), fe.getDefaultMessage()))
            .toList();
        return response;
    }

    @Getter
    @AllArgsConstructor
    public static class FieldErrorDetail {
        private final String field;
        private final String message;
    }
}
```

## 4. GlobalExceptionHandler

```java
// global/exception/GlobalExceptionHandler.java
@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    // ── 1. 모든 BusinessException (NotFoundException, ForbiddenException 등) ────
    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<ErrorResponse> handleBusiness(BusinessException e) {
        ErrorCode ec = e.getErrorCode();
        return ResponseEntity
            .status(ec.getHttpStatus())
            .body(ErrorResponse.of(ec, e.getMessage()));
    }

    // ── 2. @Valid / @Validated 유효성 실패 ──────────────────────────────────────
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ErrorResponse> handleValidation(MethodArgumentNotValidException e) {
        return ResponseEntity
            .badRequest()
            .body(ErrorResponse.ofValidation(e.getBindingResult().getFieldErrors()));
    }

    // ── 3. @RequestParam / @PathVariable 타입 불일치 ────────────────────────────
    @ExceptionHandler(MethodArgumentTypeMismatchException.class)
    public ResponseEntity<ErrorResponse> handleTypeMismatch(MethodArgumentTypeMismatchException e) {
        return ResponseEntity
            .badRequest()
            .body(ErrorResponse.of(ErrorCode.INVALID_REQUEST,
                String.format("'%s' 파라미터 값이 올바르지 않습니다.", e.getName())));
    }

    // ── 4. 지원하지 않는 HTTP 메서드 ────────────────────────────────────────────
    @ExceptionHandler(HttpRequestMethodNotSupportedException.class)
    public ResponseEntity<ErrorResponse> handleMethodNotSupported(
            HttpRequestMethodNotSupportedException e) {
        return ResponseEntity
            .status(HttpStatus.METHOD_NOT_ALLOWED)
            .body(ErrorResponse.of(ErrorCode.INVALID_REQUEST,
                String.format("'%s' 메서드는 지원하지 않습니다.", e.getMethod())));
    }

    // ── 5. DB unique constraint 위반 ────────────────────────────────────────────
    //      (USER_COUPON UNIQUE(user_id, coupon_id), ACTIVE_BOOKING PK 중복 등)
    @ExceptionHandler(DataIntegrityViolationException.class)
    public ResponseEntity<ErrorResponse> handleDataIntegrity(DataIntegrityViolationException e) {
        log.warn("DataIntegrityViolation: {}", e.getMessage());

        String message = e.getMostSpecificCause() != null
            ? e.getMostSpecificCause().getMessage() : e.getMessage();

        // USER_COUPON UNIQUE 위반 → C002로 매핑
        if (message != null && message.contains("uk_user_coupon")) {
            return ResponseEntity
                .status(HttpStatus.CONFLICT)
                .body(ErrorResponse.of(ErrorCode.COUPON_ALREADY_ISSUED));
        }

        // ACTIVE_BOOKING PK 위반 (좌석 중복 확정 최후 방어선)
        if (message != null && message.contains("active_booking")) {
            return ResponseEntity
                .status(HttpStatus.CONFLICT)
                .body(ErrorResponse.of(ErrorCode.SEAT_ALREADY_CONFIRMED));
        }

        return ResponseEntity
            .status(HttpStatus.CONFLICT)
            .body(ErrorResponse.of(ErrorCode.INVALID_REQUEST, "중복되거나 이미 처리된 요청입니다."));
    }

    // ── 6. 미처리 예외 폴백 (시스템 에러) ───────────────────────────────────────
    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleUnexpected(Exception e,
                                                          HttpServletRequest request) {
        log.error("[INTERNAL_ERROR] {} {}: {}",
            request.getMethod(), request.getRequestURI(), e.getMessage(), e);
        return ResponseEntity
            .status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(ErrorResponse.of(ErrorCode.INTERNAL_ERROR));
    }
}
```

> **Spring Security 예외 처리 위치**:
> JWT 필터에서 발생하는 인증 실패(401)는 `JwtAuthEntryPoint`에서 직접 응답 작성.
> `@ExceptionHandler`는 디스패처 서블릿 이후 단계이므로 Security 필터 예외를 잡을 수 없음.
>
> ```java
> // global/security/JwtAuthEntryPoint.java
> @Component
> public class JwtAuthEntryPoint implements AuthenticationEntryPoint {
>     @Override
>     public void commence(HttpServletRequest request, HttpServletResponse response,
>                          AuthenticationException e) throws IOException {
>         response.setContentType(MediaType.APPLICATION_JSON_VALUE);
>         response.setStatus(HttpStatus.UNAUTHORIZED.value());
>         response.getWriter().write("""
>             {"code":"A003","message":"인증이 필요합니다.","timestamp":"%s"}
>             """.formatted(LocalDateTime.now()));
>     }
> }
> ```

## 5. 도메인별 에러 발생 상황 매핑

### Auth

| 상황 | throw | ErrorCode |
|------|-------|-----------|
| 이메일 중복 가입 | `ConflictException(DUPLICATE_EMAIL)` | 409 A001 |
| 비밀번호 불일치 | Spring Security 처리 | 401 A002 |
| JWT 만료 | `JwtAuthEntryPoint` | 401 A004 |
| JWT 형식 오류 | `JwtAuthEntryPoint` | 401 A005 |

### User

| 상황 | throw | ErrorCode |
|------|-------|-----------|
| 회원 미존재 | `NotFoundException(USER_NOT_FOUND)` | 404 U001 |
| 타인 정보 수정 시도 | `ForbiddenException(USER_FORBIDDEN)` | 403 U002 |

### Event

| 상황 | throw | ErrorCode |
|------|-------|-----------|
| 이벤트 미존재 | `NotFoundException(EVENT_NOT_FOUND)` | 404 E001 |
| ADMIN 아닌데 이벤트 등록 | `ForbiddenException(ADMIN_ONLY)` | 403 G004 |
| saleStartAt ≥ eventDate | `InvalidRequestException(INVALID_SALE_DATE)` | 400 E003 |
| 구역 미존재 | `NotFoundException(SECTION_NOT_FOUND)` | 404 E004 |
| 좌석 미존재 | `NotFoundException(SEAT_NOT_FOUND)` | 404 E005 |

### Hold / Seat

| 상황 | throw | ErrorCode |
|------|-------|-----------|
| 1인 4석 초과 | `InvalidRequestException(HOLD_LIMIT_EXCEEDED)` | 400 S001 |

| hold 키 이미 존재 (ON_HOLD) | `ConflictException(SEAT_ALREADY_HELD)` | 409 S003 |
| ACTIVE_BOOKING 이미 존재 | `ConflictException(SEAT_ALREADY_CONFIRMED)` | 409 S004 |

### Order / Booking

@Transactional
public HoldResponseDto processHold(Long eventId, Long seatId, Long userId) {

    // 1. 어뷰징 체크 (최대 4석)
    Integer holdCount = (Integer) redisTemplate.opsForValue()
        .get("user-hold-count:" + userId);
    if (holdCount != null && holdCount >= 4) {
        throw new InvalidRequestException(ErrorCode.HOLD_LIMIT_EXCEEDED);
    }

    // 2. CONFIRMED 여부 확인
    if (activeBookingRepository.existsBySeatId(seatId)) {
        throw new ConflictException(ErrorCode.SEAT_ALREADY_CONFIRMED);
    }

    // 3. ON_HOLD 여부 확인
    if (Boolean.TRUE.equals(redisTemplate.hasKey("hold:" + eventId + ":" + seatId))) {
        throw new ConflictException(ErrorCode.SEAT_ALREADY_HELD);
    }

    // 4. Redis Hold 키 저장 ...
}

// OrderService.java
@Transactional
public void cancelOrder(Long orderId, Long userId) {
    Order order = orderRepository.findById(orderId)
        .orElseThrow(() -> new NotFoundException(ErrorCode.ORDER_NOT_FOUND));

    if (!order.getUserId().equals(userId)) {
        throw new ForbiddenException(ErrorCode.CANCEL_NOT_OWNED);
    }

    if (order.getStatus() == OrderStatus.CANCELLED) {
        throw new InvalidRequestException(ErrorCode.ORDER_ALREADY_CANCELLED);
    }

    // 이벤트 시작 24시간 이전인지 확인 (KST 기준)
    if (!isWithinCancelPeriod(order)) {
        throw new InvalidRequestException(ErrorCode.CANCEL_PERIOD_EXPIRED);
    }

    // SELECT FOR UPDATE 처리 ...
}

public void issueCoupon(Long couponId, Long userId) {

    // 1. 중복 발급 체크 (락 없이 — 빠른 실패)
    if (userCouponRepository.existsByUserIdAndCouponId(userId, couponId)) {

}

@Transactional
public void processWebhook(WebhookRequestDto dto) {

## 8. 패키지 구조 반영

```
global/
├── exception/
│   ├── ErrorCode.java                   ← enum (이 문서 기준, 39개 에러코드)
│   ├── GlobalExceptionHandler.java      ← @RestControllerAdvice
│   ├── BusinessException.java           ← 최상위 (ErrorCode 보유)
│   ├── NotFoundException.java           → 404
│   ├── InvalidRequestException.java     → 400
│   ├── ForbiddenException.java          → 403
│   ├── ConflictException.java           → 409
│   └── HoldExpiredException.java        → 409 (Hold TTL 만료 전용)
└── common/
    ├── ApiResponse.java                 ← 공통 성공 응답 래퍼 { data, code, message }
    └── ErrorResponse.java               ← 에러 응답 DTO { code, message, timestamp, errors[]? }
```

### ApiResponse 구조

```java
// global/common/ApiResponse.java
@Getter
@RequiredArgsConstructor
public class ApiResponse<T> {
    private final T data;
    private final String code;
    private final String message;

    public static <T> ApiResponse<T> ok(T data) {
        return new ApiResponse<>(data, "200", "OK");
    }

    public static <T> ApiResponse<T> created(T data) {
        return new ApiResponse<>(data, "201", "Created");
    }
}
```

**성공 응답 JSON 형태**:

```json
{
  "data": { "userId": 1, "email": "user@example.com" },
  "code": "200",
  "message": "OK"
}
```

> **ApiResponse vs ErrorResponse 비교**:
>
> | 구분 | 클래스 | 필드 | 사용 시점 |
> |------|--------|------|----------|
> | 성공 | `ApiResponse<T>` | `data`, `code`, `message` | 2xx 응답 |
> | 실패 | `ErrorResponse` | `code`, `message`, `timestamp`, `errors[]`(선택) | 4xx / 5xx 응답 |
>
> `status` 필드는 존재하지 않음. HTTP 상태 코드는 `ResponseEntity`의 status로만 전달.
